# Guia de Contribuição para o Speew

Ficamos muito felizes por você ter interesse em contribuir com o Speew! Toda contribuição é bem-vinda e apreciada. Este guia apresenta as diretrizes para contribuir com o projeto.

## Como Contribuir

Existem várias maneiras de contribuir, não apenas com código. Você pode:

*   **Reportar bugs:** Se você encontrar um bug, por favor, abra uma issue detalhando o problema.
*   **Sugerir melhorias:** Tem uma ideia para uma nova funcionalidade ou uma melhoria em algo existente? Abra uma issue para discutir.
*   **Escrever documentação:** A documentação pode sempre ser melhorada. Se você encontrar algo que não está claro ou que poderia ser mais bem explicado, sinta-se à vontade para fazer um pull request.
*   **Contribuir com código:** Se você quer corrigir um bug ou implementar uma nova funcionalidade, siga os passos abaixo.

## Reportando Bugs

Antes de reportar um bug, por favor, verifique as issues existentes para garantir que o bug ainda não foi reportado. Ao reportar um bug, inclua o máximo de detalhes possível, como:

*   Uma descrição clara e concisa do que é o bug.
*   Passos para reproduzir o bug.
*   O que você esperava que acontecesse.
*   O que realmente aconteceu.
*   Informações sobre o seu ambiente (versão do sistema operacional, versão do Flutter, etc.).

## Sugerindo Melhorias

Se você tem uma ideia para uma melhoria, abra uma issue com o template de "sugestão de funcionalidade". Descreva sua sugestão em detalhes, explicando por que ela seria útil para o projeto.

## Seu Primeiro Pull Request

Se você nunca contribuiu para um projeto open-source antes, seja bem-vindo! Aqui estão os passos para fazer seu primeiro pull request:

1.  **Faça um fork do repositório:** Clique no botão "Fork" no canto superior direito da página do repositório.
2.  **Clone o seu fork:** `git clone https://github.com/SEU-USUARIO/speew.git`
3.  **Crie um branch:** `git checkout -b minha-nova-funcionalidade`
4.  **Faça suas alterações:** Faça as alterações que você deseja no código.
5.  **Faça o commit de suas alterações:** `git commit -m 'Adiciona alguma funcionalidade'`
6.  **Faça o push para o seu branch:** `git push origin minha-nova-funcionalidade`
7.  **Abra um pull request:** Vá para o repositório original do Speew e abra um pull request.

## Guia de Estilo de Código

Para manter a consistência do código, por favor, siga as diretrizes de estilo do Dart e do Flutter. Use o `flutter format` para formatar seu código antes de fazer o commit.

Obrigado por contribuir com o Speew!
